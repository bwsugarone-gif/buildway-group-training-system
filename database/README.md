# Database — Schema Reference

This folder contains the SaaS multi-tenant schema draft for Buildway AI Core.

## Status

**Draft only.** Not for production migration. Schema will be finalised before Phase 2 deployment.

## Tables

| Table | Purpose |
|---|---|
| `tenants` | One row per client organisation |
| `users` | Staff/users belonging to a tenant |
| `api_keys` | Tenant-owned external API credentials (encrypted) |
| `knowledge_bases` | RAG document collections per tenant |
| `chat_sessions` | One session per customer conversation |
| `messages` | Individual messages within a session |
| `actions` | Follow-up action items generated by AI or staff |
| `usage_logs` | Token consumption and cost tracking per tenant |

## Design Principles

- Every table has `tenant_id` — strict tenant isolation at the data layer.
- API keys are stored as encrypted placeholders only. Real keys live in `.env` or a secrets manager.
- `usage_logs` tracks token usage per provider so cost is attributed to the correct tenant.
- Schema uses PostgreSQL (`gen_random_uuid()`, `UUID`). Adapt for SQLite in local dev if needed.

## Running the Schema

```bash
# PostgreSQL
psql -U your_user -d your_db -f schema.sql
```

## Notes

- Do not commit real API keys or credentials anywhere in this repo.
- See `.env.example` at the project root for required environment variables.
